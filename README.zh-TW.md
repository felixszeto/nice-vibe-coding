# Vibe：AI 驅動的低代碼應用程式平台

## 📄 概覽

Vibe 是一個動態的、基於 Web 的平台，讓使用者能利用人工智慧的力量來創建、管理和分享單頁 Web 應用程式。使用者只需描述他們需要的工具或介面，AI 後端便會生成所需
的 HTML、CSS 和 JavaScript 代碼。此專案旨在彌合想法與實現之間的差距，實現簡單 Web 工具的快速原型設計和部署。

平台提供一個完整的生態系統，包括安全的使用者身份驗證系統、供開發者管理其創作的工作空間、一個用於瀏覽和使用已發佈應用程式的「應用程式商店」，以及一個供管理員監督使用者、審核提交內容和配置系統的綜合管理中心。

## ✨ 主要功能

*   **對話式 AI 開發**：Vibe 的核心是一個互動式的、基於聊天的介面，使用者可以在這裡將他們的想法變為現實。只需與 AI 對話，描述您想建立的應用程式，請求變更，並逐步完善其功能。AI 會處理編碼工作，讓您能專注於工具的設計和目的。此過程完全受版本控制，因此您隨時可以恢復到先前的狀態。

*   **使用者與會話管理**：安全的登入系統，具備密碼雜湊功能。經過身份驗證的使用者擁有專屬的會話和工作空間來管理他們的應用程式。

*   **應用程式生命週期管理**：
    *   **草稿與版本控制**：每次變更都會創建一個新版本，並提供完整的歷史記錄。使用者可以恢復到任何先前的版本。
    *   **審核與批准工作流程**：開發者可以提交他們的應用程式以供審核。管理員可以根據功能和安全性批准或拒絕提交。
    *   **發佈**：經批准的應用程式可以「上線」，並將出現在公共的應用程式商店中，供所有使用者存取。

*   **應用程式商店**：一個供所有使用者發現、瀏覽和使用已發佈應用程式的中心平台。每個應用程式都包含描述、預覽和開發者資訊。

*   **綜合管理中心**：一個受權限控制的後端，供管理員和審核人員使用，功能包括：
    *   **系統儀表板**：查看關鍵統計數據，如使用者數量、應用程式狀態分佈和創建趨勢。
    *   **使用者管理**：創建、編輯和管理使用者、他們的狀態和群組成員資格。
    *   **群組與權限控制**：通過將使用者分配到具有特定權限（例如 `develop_app`、`review_app`、`system_admin`）的群組，實現細粒度的存取控制。
    *   **AI 設定**：管理 AI 模型、API 金鑰和用於不同任務（代碼生成、風險分析）的系統級提示。

*   **風險分析與內容審核**：AI 驅動的分析，用於在應用程式發佈前識別潛在的安全風險（例如，腳本注入）並對其進行分類。

*   **國際化 (i18n)**：系統支援五種語言：中文(繁體)、中文(簡體)、日文、韓文、英文。

## 🏗️ 專案結構

該專案主要由兩個主要元件組成，它們與一個資料庫和輔助模組協同工作。

*   **`main.py`**：這是應用程式的核心和主要進入點。它負責：
    *   初始化 `NiceGUI` Web 伺服器和 UI。
    *   處理所有使用者導向的頁面，如登入畫面 (`/login`)、應用程式創建/編輯介面 (`/session/{session_id}`)、應用程式商店 (`/store`)，以及個人應用程式管理頁面 (`/my-apps`, `/my-shares`)。
    *   管理 AI 互動邏輯，將使用者提示發送到 AI 服務並處理生成的 HTML。
    *   提供已發佈的應用程式和分享連結。

*   **`management_pages.py`**：此檔案定義了整個後端管理介面。它是應用程式的一個獨立部分，僅供具有適當權限（`system_admin` 或 `review_app`）的使用者存取。其職責包括：
    *   渲染帶有系統分析的管理儀表板。
    *   提供用於管理使用者、群組和權限的 UI 和邏輯。
    *   定義應用程式審核工作流程，允許管理員查看提交的應用程式、生成風險報告，並批准或拒絕它們。
    *   處理系統級設定，如 AI 模型設定和提示範本。

這兩個檔案與 `database.py`（管理 SQLite 資料庫中的所有資料持久性）和 `languages.py`（為 UI 提供翻譯服務）互動。

## 📋 先決條件

要執行此專案，您需要安裝 `requirements.txt` 中列出的 Python 庫。

## 🚀 安裝

1.  **克隆儲存庫：**
    ```bash
    git clone https://github.com/felixszeto/nice-vibe-coding.git
    cd nice-vibe-coding
    ```

2.  **安裝依賴項：**
    建議使用虛擬環境。
    ```bash
    # 創建並啟動虛擬環境（可選但建議）
    python -m venv venv
    source venv/bin/activate  # 在 Windows 上，使用 `venv\Scripts\activate`

    # 安裝所需的套件
    pip install -r requirements.txt
    ```

## 💻 使用方法

1.  **初始化資料庫：**
    首次執行應用程式時，它會自動創建一個 `app_data.db` 檔案，其中包含必要的資料表和初始資料，包括一個預設的管理員使用者。
    *   **預設管理員憑證**：
        *   使用者名稱：`admin`
        *   密碼：`admin123`
    *   **預設使用者憑證**：
        *   使用者名稱：`tester`
        *   密碼：`admin123`
        *   （此使用者具有基本權限，只能使用已發佈的應用程式。請使用 `admin` 帳戶創建具有不同角色的使用者。）

2. **預載範例：**
   資料庫預載了一些範例應用程式，以展示平台的功能。登入後，您可以在應用程式商店中找到它們：
    *   薪資計算器
    *   文件壓縮器
    *   待辦事項列表
    *   電腦模擬
    *   QR Code 生成器
    *   付款記錄器

3.  **執行應用程式：**
    執行 `main.py` 檔案以啟動 Web 伺服器。
    ```bash
    python main.py
    ```
    應用程式預設將在 `http://127.0.0.1:8462` 上提供。

4.  **使用平台：**
    *   **登入**：在瀏覽器中存取應用程式 URL，並使用管理員憑證或您創建的任何其他使用者登入。
    *   **創建應用程式**：導航至「我的應用程式」，然後點擊「新增」按鈕以開始新的創建會話。使用聊天介面描述您想建立的應用程式。
    *   **管理應用程式**：從「我的應用程式」頁面，您可以查看現有的應用程式，檢查其狀態（草稿、審核中、已發佈），並進入會話進行管理。
    *   **管理功能**：以 `admin` 身份登入，並從側邊欄導航至「管理中心」以存取所有管理功能。
