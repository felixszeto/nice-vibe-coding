# Vibe: AI 기반 로우코드 애플리케이션 플랫폼

## 📄 개요

Vibe는 사용자가 인공지능의 힘을 이용해 단일 페이지 웹 애플리케이션을 만들고, 관리하고, 공유할 수 있는 동적 웹 기반 플랫폼입니다. 사용자가 필요한 도구나 인터페이스를 설명하기만 하면 AI 백엔드가 필요한 HTML, CSS, JavaScript 코드를 생성합니다. 이 프로젝트는 아이디어와 구현 사이의 격차를 해소하여 간단한 웹 도구의 신속한 프로토타이핑과 배포를 가능하게 하도록 설계되었습니다.

플랫폼은 안전한 사용자 인증 시스템, 개발자가 자신의 창작물을 관리할 수 있는 작업 공간, 게시된 애플리케이션을 탐색하고 사용할 수 있는 '앱 스토어', 그리고 관리자가 사용자를 감독하고, 제출물을 검토하며, 시스템을 구성할 수 있는 포괄적인 관리 센터를 포함한 완전한 생태계를 갖추고 있습니다.

## ✨ 주요 기능

*   **대화형 AI 개발**: Vibe의 핵심은 사용자가 아이디어를 현실로 만들 수 있는 대화형, 채팅 기반 인터페이스입니다. AI와 대화하며 만들고 싶은 애플리케이션을 설명하고, 변경을 요청하며, 기능을 점진적으로 개선할 수 있습니다. AI가 코딩을 처리하므로 사용자는 도구의 디자인과 목적에 집중할 수 있습니다. 이 프로세스는 완전히 버전 제어되므로 언제든지 이전 상태로 돌아갈 수 있습니다.

*   **사용자 및 세션 관리**: 비밀번호 해싱 기능이 있는 안전한 로그인 시스템. 인증된 사용자는 자신의 애플리케이션을 관리할 수 있는 전용 세션과 작업 공간을 가집니다.

*   **애플리케이션 수명 주기 관리**:
    *   **초안 및 버전 관리**: 모든 변경 사항은 새 버전을 생성하며, 전체 기록을 볼 수 있습니다. 사용자는 이전 버전으로 되돌릴 수 있습니다.
    *   **검토 및 승인 워크플로**: 개발자는 검토를 위해 애플리케이션을 제출할 수 있습니다. 관리자는 기능 및 안전성을 기준으로 제출을 승인하거나 거부할 수 있습니다.
    *   **게시**: 승인된 애플리케이션은 '라이브' 상태가 되어 모든 사용자가 접근할 수 있는 공개 앱 스토어에 표시됩니다.

*   **앱 스토어**: 모든 사용자가 게시된 애플리케이션을 발견하고, 탐색하며, 사용할 수 있는 중앙 허브입니다. 각 앱에는 설명, 미리보기, 개발자 정보가 포함됩니다.

*   **포괄적인 관리 센터**: 관리자 및 검토자를 위한 권한 제어 백엔드로, 다음 기능을 제공합니다:
    *   **시스템 대시보드**: 사용자 수, 앱 상태 분포, 생성 추세 등 주요 통계를 봅니다.
    *   **사용자 관리**: 사용자, 상태, 그룹 멤버십을 생성, 편집, 관리합니다.
    *   **그룹 및 권한 제어**: 사용자를 특정 권한(`develop_app`, `review_app`, `system_admin` 등)을 가진 그룹에 할당하여 세분화된 접근 제어를 구현합니다.
    *   **AI 구성**: AI 모델, API 키 및 다양한 작업(코드 생성, 위험 분석)을 위한 시스템 수준 프롬프트를 관리합니다.

*   **위험 분석 및 콘텐츠 조정**: AI 기반 분석을 통해 게시 전에 잠재적인 보안 위험(예: 스크립트 주입)을 식별하고 애플리케이션을 분류합니다.

*   **국제화(i18n)**: 시스템은 영어, 중국어(번체), 중국어(간체), 일본어, 한국어의 5개 언어를 지원합니다.

## 🏗️ 프로젝트 구조

이 프로젝트는 주로 두 개의 주요 구성 요소로 구성되며, 데이터베이스 및 헬퍼 모듈과 함께 작동합니다.

*   **`main.py`**: 애플리케이션의 핵심이자 메인 진입점입니다. 다음을 담당합니다:
    *   `NiceGUI` 웹 서버 및 UI 초기화.
    *   로그인 화면(`/login`), 앱 생성/편집 인터페이스(`/session/{session_id}`), 앱 스토어(`/store`), 개인 앱 관리 페이지(`/my-apps`, `/my-shares`) 등 모든 사용자 대상 페이지 처리.
    *   AI 상호 작용 로직 관리, 사용자 프롬프트를 AI 서비스로 보내고 생성된 HTML 처리.
    *   게시된 애플리케이션 및 공유 링크 제공.

*   **`management_pages.py`**: 전체 백엔드 관리 인터페이스를 정의하는 파일입니다. 적절한 권한(`system_admin` 또는 `review_app`)을 가진 사용자만 접근할 수 있는 애플리케이션의 별도 섹션입니다. 담당 업무는 다음과 같습니다:
    *   시스템 분석이 포함된 관리 대시보드 렌더링.
    *   사용자, 그룹, 권한 관리를 위한 UI 및 로직 제공.
    *   관리자가 제출된 앱을 보고, 위험 보고서를 생성하며, 승인 또는 거부할 수 있는 앱 검토 워크플로 정의.
    *   AI 모델 설정 및 프롬프트 템플릿과 같은 시스템 수준 구성 처리.

이 두 파일은 SQLite 데이터베이스의 모든 데이터 지속성을 관리하는 `database.py`와 UI에 번역 서비스를 제공하는 `languages.py`와 상호 작용합니다.

## 📋 전제 조건

이 프로젝트를 실행하려면 `requirements.txt`에 나열된 Python 라이브러리를 설치해야 합니다.

## 🚀 설치

1.  **리포지토리 복제:**
    ```bash
    git clone https://github.com/felixszeto/nice-vibe-coding.git
    cd nice-vibe-coding
    ```

2.  **의존성 설치:**
    가상 환경 사용을 권장합니다.
    ```bash
    # 가상 환경 생성 및 활성화 (선택 사항이지만 권장)
    python -m venv venv
    source venv/bin/activate  # Windows에서는 `venv\Scripts\activate` 사용

    # 필요한 패키지 설치
    pip install -r requirements.txt
    ```

## 💻 사용법

1.  **데이터베이스 초기화:**
    애플리케이션을 처음 실행하면 필요한 테이블과 초기 데이터(기본 관리자 사용자 포함)가 포함된 `app_data.db` 파일이 자동으로 생성됩니다.
    *   **기본 관리자 자격 증명**:
        *   사용자 이름: `admin`
        *   비밀번호: `admin123`
    *   **기본 사용자 자격 증명**:
        *   사용자 이름: `tester`
        *   비밀번호: `admin123`
        *   (이 사용자는 기본 권한만 가지고 있으며 게시된 애플리케이션만 사용할 수 있습니다. 다른 역할을 가진 사용자를 만들려면 `admin` 계정을 사용하세요.)

2. **사전 로드된 예제:**
   데이터베이스에는 플랫폼의 기능을 보여주는 몇 가지 예제 애플리케이션이 사전 로드되어 있습니다. 로그인 후 앱 스토어에서 찾을 수 있습니다:
    *   급여 계산기
    *   문서 압축기
    *   할 일 목록
    *   컴퓨터 시뮬레이션
    *   QR 코드 생성기
    *   결제 기록기

3.  **애플리케이션 실행:**
    `main.py` 파일을 실행하여 웹 서버를 시작합니다.
    ```bash
    python main.py
    ```
    애플리케이션은 기본적으로 `http://127.0.0.1:8462`에서 사용할 수 있습니다.

4.  **플랫폼 사용:**
    *   **로그인**: 브라우저에서 애플리케이션 URL에 접속하여 관리자 자격 증명이나 직접 만든 다른 사용자로 로그인합니다.
    *   **앱 만들기**: '내 앱'으로 이동하여 '추가' 버튼을 클릭하여 새 생성 세션을 시작합니다. 채팅 인터페이스를 사용하여 만들고 싶은 애플리케이션을 설명합니다.
    *   **앱 관리**: '내 앱' 페이지에서 기존 애플리케이션을 보고 상태(초안, 검토 중, 게시됨)를 확인하고 세션에 들어가 관리할 수 있습니다.
    *   **관리자 기능**: `admin`으로 로그인하고 사이드 드로어에서 '관리 센터'로 이동하여 모든 관리 기능에 접근합니다.
