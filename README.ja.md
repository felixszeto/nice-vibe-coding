# Vibe：AI搭載ローコードアプリケーションプラットフォーム

## 📄 概要

Vibeは、人工知能の力を利用してユーザーがシングルページのWebアプリケーションを作成、管理、共有できる、動的なWebベースのプラットフォームです。ユーザーが必要なツールやインターフェースを説明するだけで、AIバックエンドが必要なHTML、CSS、JavaScriptコードを生成します。このプロジェクトは、アイデアと実装の間のギャップを埋め、シンプルなWebツールの迅速なプロトタイピングと展開を可能にすることを目的としています。

プラットフォームは、安全なユーザー認証システム、開発者が自身の作品を管理するためのワークスペース、公開されたアプリケーションを閲覧・使用するための「アプリストア」、管理者がユーザーを監督し、提出物を確認し、システムを設定するための包括的な管理センターを含む、完全なエコシステムを備えています。

## ✨ 主な機能

*   **対話型AI開発**：Vibeの中核は、ユーザーがアイデアを形にするインタラクティブなチャットベースのインターフェースです。AIと対話するだけで、構築したいアプリケーションを説明し、変更を要求し、その機能を段階的に改良できます。AIがコーディングを担当するため、ユーザーはツールの設計と目的に集中できます。このプロセスは完全にバージョン管理されており、いつでも以前の状態に戻すことができます。

*   **ユーザーとセッション管理**：パスワードハッシュ機能を備えた安全なログインシステム。認証されたユーザーは、アプリケーションを管理するための専用のセッションとワークスペースを持ちます。

*   **アプリケーションライフサイクル管理**：
    *   **ドラフトとバージョン管理**：変更のたびに新しいバージョンが作成され、完全な履歴が利用可能です。ユーザーは任意の以前のバージョンに戻すことができます。
    *   **レビューと承認ワークフロー**：開発者はレビューのためにアプリケーションを提出できます。管理者は機能性と安全性に基づいて提出を承認または拒否できます。
    *   **公開**：承認されたアプリケーションは「ライブ」になり、すべてのユーザーがアクセスできる公開アプリストアに表示されます。

*   **アプリストア**：すべてのユーザーが公開されたアプリケーションを発見、閲覧、使用できる中央の場所。各アプリには説明、プレビュー、開発者情報が含まれています。

*   **包括的な管理センター**：管理者とレビュワー向けの権限制御されたバックエンドで、以下の機能を備えています：
    *   **システムダッシュボード**：ユーザー数、アプリのステータス分布、作成トレンドなどの主要な統計情報を表示します。
    *   **ユーザー管理**：ユーザー、そのステータス、グループメンバーシップを作成、編集、管理します。
    *   **グループと権限管理**：ユーザーを特定の権限（例：`develop_app`、`review_app`、`system_admin`）を持つグループに割り当てることで、きめ細かいアクセス制御を実現します。
    *   **AI設定**：AIモデル、APIキー、およびさまざまなタスク（コード生成、リスク分析）のためのシステムレベルのプロンプトを管理します。

*   **リスク分析とコンテンツモデレーション**：AIによる分析で、公開前に潜在的なセキュリティリスク（例：スクリプトインジェクション）を特定し、アプリケーションを分類します。

*   **国際化（i18n）**：システムは英語、中国語（繁体字）、中国語（簡体字）、日本語、韓国語の5つの言語をサポートしています。

## 🏗️ プロジェクト構造

このプロジェクトは主に2つの主要コンポーネントで構成されており、これらがデータベースおよびヘルパーモジュールと連携して動作します。

*   **`main.py`**：これはアプリケーションの中核であり、メインのエントリポイントとして機能します。担当は次のとおりです：
    *   `NiceGUI` WebサーバーとUIの初期化。
    *   ログイン画面（`/login`）、アプリ作成/編集インターフェース（`/session/{session_id}`）、アプリストア（`/store`）、個人アプリ管理ページ（`/my-apps`、`/my-shares`）など、すべてのユーザー向けページの処理。
    *   AIインタラクションロジックの管理、ユーザープロンプトをAIサービスに送信し、生成されたHTMLを処理。
    *   公開されたアプリケーションと共有リンクの提供。

*   **`management_pages.py`**：このファイルは、バックエンド管理インターフェース全体を定義します。これはアプリケーションの別個のセクションであり、適切な権限（`system_admin`または`review_app`）を持つユーザーのみがアクセスできます。その責任は次のとおりです：
    *   システム分析付きの管理ダッシュボードのレンダリング。
    *   ユーザー、グループ、権限を管理するためのUIとロジックの提供。
    *   アプリレビューワークフローの定義。これにより、管理者は提出されたアプリを表示し、リスクレポートを生成し、それらを承認または拒否できます。
    *   AIモデル設定やプロンプトテンプレートなどのシステムレベルの設定の処理。

これら2つのファイルは、SQLiteデータベースのすべてのデータ永続性を管理する`database.py`と、UIに翻訳サービスを提供する`languages.py`と対話します。

## 📋 前提条件

このプロジェクトを実行するには、`requirements.txt`にリストされているPythonライブラリをインストールする必要があります。

## 🚀 インストール

1.  **リポジトリのクローン：**
    ```bash
    git clone https://github.com/felixszeto/nice-vibe-coding.git
    cd nice-vibe-coding
    ```

2.  **依存関係のインストール：**
    仮想環境の使用をお勧めします。
    ```bash
    # 仮想環境の作成とアクティベート（任意ですが推奨）
    python -m venv venv
    source venv/bin/activate  # Windowsでは `venv\Scripts\activate` を使用

    # 必要なパッケージのインストール
    pip install -r requirements.txt
    ```

## 💻 使用方法

1.  **データベースの初期化：**
    アプリケーションを初めて実行すると、必要なテーブルと初期データ（デフォルトの管理者ユーザーを含む）を持つ`app_data.db`ファイルが自動的に作成されます。
    *   **デフォルトの管理者認証情報**：
        *   ユーザー名：`admin`
        *   パスワード：`admin123`
    *   **デフォルトのユーザー認証情報**：
        *   ユーザー名：`tester`
        *   パスワード：`admin123`
        *   （このユーザーは基本的な権限のみを持ち、公開されたアプリケーションしか使用できません。異なるロールを持つユーザーを作成するには、`admin`アカウントを使用してください。）

2. **プリロードされた例：**
   データベースには、プラットフォームの機能を示すいくつかのサンプルアプリケーションがプリロードされています。ログイン後、アプリストアでそれらを見つけることができます：
    *   給与計算機
    *   ドキュメント圧縮機
    *   To-Doリスト
    *   コンピューターシミュレーション
    *   QRコードジェネレーター
    *   支払いレコーダー

3.  **アプリケーションの実行：**
    `main.py`ファイルを実行してWebサーバーを起動します。
    ```bash
    python main.py
    ```
    アプリケーションはデフォルトで`http://127.0.0.1:8462`で利用可能になります。

4.  **プラットフォームの使用：**
    *   **ログイン**：ブラウザでアプリケーションのURLにアクセスし、管理者認証情報または作成した他のユーザーでログインします。
    *   **アプリの作成**：「マイアプリ」に移動し、「追加」ボタンをクリックして新しい作成セッションを開始します。チャットインターフェースを使用して、構築したいアプリケーションを説明します。
    *   **アプリの管理**：「マイアプリ」ページから、既存のアプリケーションを表示し、そのステータス（ドラフト、レビュー中、公開済み）を確認し、セッションに入って管理できます。
    *   **管理者機能**： `admin`としてログインし、サイドドロワーから「管理センター」に移動して、すべての管理機能にアクセスします。
